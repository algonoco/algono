// what it targets

Six patterns. One toolkit.

  • Stale Domain Admin / Global Admin accounts
    No MFA, no recent logon, no documented owner — high blast radius
    Critical
  • Service accounts with interactive login rights
    Can log in interactively, often have password age > 365 days
    Critical
  • Nested group privilege escalation paths
    Unprivileged group → nested into admin group → effective Domain Admin
    High
  • Entra app registrations with owner-level Graph permissions
    AppRoles or delegated permissions at owner/admin scope, no expiry
    High
  • Orphaned guest accounts with directory access
    Directory.Read or above, external identity, no recent sign-in
    High
  • Managed identity over-scope
    Azure managed identities with Contributor or Owner at subscription scope
    Medium

// what you get

Scan, remediate, roll back.

[01]
Audit Script — read-only
PowerShell. Connects to AD and Entra ID (Graph API). Produces a structured JSON findings report and a terminal summary. No writes. Safe to run in production.

Remediation Script
Applies fixes from the findings report. Every write operation is captured in a timestamped rollback manifest before execution. Supports dry-run mode. Scoped — only touches what the audit flagged.
[03]
Rollback Script
Reads the manifest. Undoes every write. Restores group memberships, account states, and permission scopes to pre-remediation state. If something breaks post-fix, you're one command out.
